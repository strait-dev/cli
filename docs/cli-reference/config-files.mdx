## Global config (`~/.config/strait/config.yaml`)

Your user-level CLI configuration. Manages contexts, aliases, and defaults.

```yaml
# Active context name
active_context: prod

# Named contexts for different environments
contexts:
  prod:
    server: https://api.strait.dev
    project: proj_abc123
    format: table
  staging:
    server: https://staging.strait.dev
    project: proj_def456
    format: json

# Command aliases
aliases:
  trig: trigger
  rl: "runs list --status failed"
  wl: "workflows list"

# Default values (used when no context is active)
server: https://api.strait.dev
project: proj_abc123
format: table
```

### Fields

| Field | Type | Description |
|---|---|---|
| `active_context` | string | Name of the currently active context |
| `contexts` | map | Named context configurations |
| `contexts.<name>.server` | string | API server URL |
| `contexts.<name>.project` | string | Default project ID |
| `contexts.<name>.format` | string | Default output format |
| `aliases` | map | Command alias mappings |
| `server` | string | Fallback server URL |
| `project` | string | Fallback project ID |
| `format` | string | Fallback output format |
| `api_key` | string | Fallback API key (prefer keyring) |

<Warning>
  Avoid storing `api_key` in config files. Use `strait auth login` to store credentials securely in the system keyring.
</Warning>

## Local project config (`.strait.yaml`)

Project-level overrides in the current directory. Same structure as the global config but applies only when running from that directory.

```yaml
server: https://api.strait.dev
project: proj_abc123
format: json
```

<Info>
  If a local config contains sensitive fields like `api_key`, the CLI warns you on each command.
</Info>

## Project orchestration file (`strait.json`)

Defines the jobs and workflows Strait orchestrates. It does not describe how
your code is built, hosted, or deployed.

```json
{
  "$schema": "https://schemas.strait.dev/v1/strait.json",
  "version": "1",
  "jobs": [
    {
      "slug": "sync-inventory",
      "name": "Sync Inventory",
      "endpoint_url": "https://api.example.com/jobs/sync",
      "cron": "*/10 * * * *",
      "timeout_secs": 120,
      "max_attempts": 3
    }
  ],
  "workflows": [
    {
      "slug": "data-pipeline",
      "name": "Data Pipeline",
      "steps": [
        {
          "step_ref": "extract",
          "job_ref": "extract-data"
        }
      ]
    }
  ]
}
```

### Fields

| Field | Type | Required | Description |
|---|---|---|---|
| `$schema` | string | No | JSON schema URL |
| `version` | string | Yes | File format version |
| `jobs[]` | array | No | Job definitions |
| `jobs[].slug` | string | Yes | Unique job slug |
| `jobs[].name` | string | No | Display name |
| `jobs[].endpoint_url` | string | Yes | HTTP endpoint URL |
| `jobs[].cron` | string | No | Cron schedule expression |
| `jobs[].timeout_secs` | int | No | Execution timeout |
| `jobs[].max_attempts` | int | No | Max retries |
| `jobs[].description` | string | No | Description |
| `workflows[]` | array | No | Workflow definitions |
| `workflows[].slug` | string | Yes | Unique workflow slug |
| `workflows[].name` | string | No | Display name |
| `workflows[].steps[]` | array | No | Workflow steps |
| `workflows[].steps[].step_ref` | string | Yes | Step reference name |
| `workflows[].steps[].job_ref` | string | Yes | Job slug this step runs |
| `workflows[].steps[].depends_on` | array | No | Step refs this step depends on |
| `workflows[].steps[].on_failure` | string | No | Failure behavior (e.g. `continue`) |
