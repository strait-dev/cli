## Global config (`~/.config/strait/config.yaml`)

Your user-level CLI configuration. Manages contexts, aliases, and defaults.

```yaml
# Active context name
active_context: prod

# Named contexts for different environments
contexts:
  prod:
    server: https://api.strait.dev
    project: proj_abc123
    format: table
  staging:
    server: https://staging.strait.dev
    project: proj_def456
    format: json

# Command aliases
aliases:
  trig: trigger
  rl: "runs list --status failed"
  wl: "workflows list"

# Default values (used when no context is active)
server: https://api.strait.dev
project: proj_abc123
format: table
```

### Fields

| Field | Type | Description |
|---|---|---|
| `active_context` | string | Name of the currently active context |
| `contexts` | map | Named context configurations |
| `contexts.<name>.server` | string | API server URL |
| `contexts.<name>.project` | string | Default project ID |
| `contexts.<name>.format` | string | Default output format |
| `aliases` | map | Command alias mappings |
| `server` | string | Fallback server URL |
| `project` | string | Fallback project ID |
| `format` | string | Fallback output format |
| `api_key` | string | Fallback API key (prefer keyring) |

<Warning>
  Avoid storing `api_key` in config files. Use `strait login` to store credentials securely in the system keyring.
</Warning>

## Local project config (`.strait.yaml`)

Project-level overrides in the current directory. Same structure as the global config but applies only when running from that directory.

```yaml
server: https://api.strait.dev
project: proj_abc123
format: json
```

<Info>
  If a local config contains sensitive fields like `api_key`, the CLI warns you on each command.
</Info>

## Project manifest (`strait.config.json` / `strait.config.yaml`)

Defines your project's jobs and workflows for `strait build`, `strait deploy --config`, and `strait init`.

<Tabs>
  <Tab title="JSON">
    ```json
    {
      "project": {
        "id": "proj_abc123",
        "name": "My Project"
      },
      "runtime": "node",
      "build": {
        "outDir": ".strait"
      },
      "deploy": {
        "defaultEnvironment": "production"
      },
      "jobs": [
        {
          "slug": "sync-inventory",
          "name": "Sync Inventory",
          "endpointUrl": "https://api.example.com/jobs/sync",
          "cron": "*/10 * * * *",
          "timeoutSecs": 120,
          "maxAttempts": 3
        },
        {
          "slug": "send-reports",
          "name": "Send Reports",
          "endpointUrl": "https://api.example.com/jobs/reports",
          "cron": "0 9 * * 1"
        }
      ],
      "workflows": [
        {
          "slug": "data-pipeline",
          "name": "Data Pipeline",
          "steps": [
            {
              "stepRef": "extract",
              "jobRef": "extract-data"
            },
            {
              "stepRef": "transform",
              "jobRef": "transform-data",
              "dependsOn": ["extract"]
            },
            {
              "stepRef": "load",
              "jobRef": "load-data",
              "dependsOn": ["transform"],
              "onFailure": "continue"
            }
          ]
        }
      ]
    }
    ```
  </Tab>

  <Tab title="YAML">
    ```yaml
    project:
      id: proj_abc123
      name: My Project
    runtime: node
    build:
      outDir: .strait
    deploy:
      defaultEnvironment: production
    jobs:
      - slug: sync-inventory
        name: Sync Inventory
        endpointUrl: https://api.example.com/jobs/sync
        cron: "*/10 * * * *"
        timeoutSecs: 120
        maxAttempts: 3
      - slug: send-reports
        name: Send Reports
        endpointUrl: https://api.example.com/jobs/reports
        cron: "0 9 * * 1"
    workflows:
      - slug: data-pipeline
        name: Data Pipeline
        steps:
          - stepRef: extract
            jobRef: extract-data
          - stepRef: transform
            jobRef: transform-data
            dependsOn: [extract]
          - stepRef: load
            jobRef: load-data
            dependsOn: [transform]
            onFailure: continue
    ```
  </Tab>
</Tabs>

### Fields

| Field | Type | Required | Description |
|---|---|---|---|
| `project.id` | string | Yes | Project ID |
| `project.name` | string | No | Display name |
| `runtime` | string | No | Runtime hint (e.g. `node`, `go`, `python`) |
| `build.outDir` | string | No | Output directory for compiled manifests |
| `deploy.defaultEnvironment` | string | No | Default deployment environment |
| `jobs[]` | array | No | Job definitions |
| `jobs[].slug` | string | Yes | Unique job slug |
| `jobs[].name` | string | Yes | Display name |
| `jobs[].endpointUrl` | string | No | HTTP endpoint URL |
| `jobs[].cron` | string | No | Cron schedule expression |
| `jobs[].timeoutSecs` | int | No | Execution timeout (default: 60) |
| `jobs[].maxAttempts` | int | No | Max retries (default: 3) |
| `jobs[].description` | string | No | Description |
| `workflows[]` | array | No | Workflow definitions |
| `workflows[].slug` | string | Yes | Unique workflow slug |
| `workflows[].name` | string | Yes | Display name |
| `workflows[].steps[]` | array | No | Workflow steps |
| `workflows[].steps[].stepRef` | string | Yes | Step reference name |
| `workflows[].steps[].jobRef` | string | Yes | Job slug this step runs |
| `workflows[].steps[].dependsOn` | array | No | Step refs this step depends on |
| `workflows[].steps[].onFailure` | string | No | Failure behavior (e.g. `continue`) |
