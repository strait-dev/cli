This path logs you in, scaffolds a project, syncs its orchestration
definitions, triggers a job, and watches it run.

<Steps>
  <Step title="Log in">
    ```bash
    strait auth login
    ```

    Your browser opens, you confirm a one-time code, and you are in. The API key is stored in your system keyring — you will not need to paste it again.
  </Step>

  <Step title="Confirm you are authenticated">
    ```bash
    strait auth whoami
    ```

    You should see your user, context, server, and project.
  </Step>

  <Step title="Set up your project">
    ```bash
    strait init --template vercel --name my-app
    cd my-app
    ```

    This scaffolds a starter runtime and a canonical `strait.json` orchestration file.
  </Step>

  <Step title="Sync orchestration definitions">
    ```bash
    strait sync
    ```

    `strait sync` registers jobs and workflows from `strait.json`. Your code still runs on your infrastructure.
  </Step>

  <Step title="Trigger it">
    ```bash
    strait jobs trigger hello --payload '{"hello": "world"}'
    ```

    The CLI returns a run ID. Hold on to it for the next step.
  </Step>

  <Step title="Watch it run">
    ```bash
    strait runs watch <run-id>
    ```

    The CLI polls until the run completes (or fails). Want the full log stream instead? Use `strait runs logs <run-id> --follow`.
  </Step>
</Steps>
