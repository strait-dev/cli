## Start dev mode

Spin up Docker dependencies and a local runtime:

```bash
strait dev
```

| Flag | What it does |
|---|---|
| `--no-docker` | Skip Docker — useful if you manage your own services |
| `--port` | Override the default port |
| `--seed` | Populate with sample data |

```bash
strait dev --no-docker --port 9090
strait dev --seed
```

## Test your job handlers

Run a job handler locally without deploying:

```bash
strait dev test process-payment --payload '{"id": "123"}'
```

Test all jobs defined in your config:

```bash
strait dev test --all --config strait.config.json
```

## Tunnel to your local machine

Need Strait to reach a local endpoint during development? Open a Cloudflare tunnel:

```bash
strait dev tunnel --port 3000
```

This creates a public URL that routes to `localhost:3000`. Use it as the job endpoint while iterating.

## Check dev status

```bash
strait dev status
```

## Run commands with context

Inject Strait environment variables into any local command:

```bash
strait run -- node worker.js
strait run --context staging -- python process.py
```

This passes `STRAIT_SERVER`, `STRAIT_API_KEY`, `STRAIT_PROJECT`, and other context variables to the child process.
