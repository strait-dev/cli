---
title: "Sync Orchestration Definitions"
description: "Use strait.json to register jobs and workflows while your code runs on your infrastructure"
---

Strait is orchestration-only. It does not build, host, deploy, or run your
application code. Deploy your HTTP handler or worker to your own platform, then
sync the orchestration metadata Strait needs.

## 1. Deploy your runtime

Use the platform that owns your code:

```bash
vercel deploy
wrangler deploy
netlify deploy --prod
kubectl apply -f k8s/deployment.yaml
```

## 2. Set the endpoint

For HTTPS push jobs, point Strait at the route that mounts your signed serve
handler:

```bash
strait endpoint set hello https://my-app.example.com/api/strait
strait endpoint verify hello
```

## 3. Sync strait.json

`strait.json` is the project orchestration file.

```bash
strait sync
strait sync --dry-run
strait sync --prune --yes
```

Use `--file` when the file is not in the current directory:

```bash
strait sync --file ./configs/strait.json
```

## strait.json

```json
{
  "$schema": "https://schemas.strait.dev/v1/strait.json",
  "version": "1",
  "jobs": [
    {
      "slug": "hello",
      "name": "Hello",
      "endpoint_url": "https://my-app.example.com/api/strait",
      "max_attempts": 3,
      "timeout_secs": 30
    }
  ],
  "workflows": []
}
```

## Compatibility

`strait deploy push` is deprecated. During the transition it delegates to
`strait sync`, but new scripts should use `strait sync`.
